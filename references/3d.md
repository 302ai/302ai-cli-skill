# 3D Module

Text/image-to-3D model generation (**async**). Supports two backends: Hyper3D Rodin (text & image) and Doubao Seed3D (image only).

## 3d create

Create a 3D generation task.

```bash
302ai 3d create [OPTIONS]
```

### Options

| Option | Type | Required | Default | Notes |
|--------|------|----------|---------|-------|
| `--prompt` | string | t23d only | — | Text description |
| `--model` | string | No | auto-inferred | Model name; auto-selects `t23d`/`i23d` default if omitted |
| `--image` | string | i23d only | — | Repeatable; online image URL. `doubao-seed3d` accepts exactly 1 |
| `--fileformat` | string | No | — | Output format: `glb`, `usdz`, `fbx`, `obj`, `stl` |
| `--material` | string | No | — | `PBR`, `Shaded` (hyper3d-rodin); `PBR`, `Shaded`, `All`, `None` (doubao-seed3d) |
| `--quality` | string | No | — | `high`, `medium`, `low`, `extra-low` (hyper3d-rodin only) |
| `--tier` | string | No | — | `Regular`, `Sketch` (hyper3d-rodin only) |
| `--mesh_mode` | string | No | — | `Raw`, `Quad` (doubao-seed3d only) |
| `--subdivisionlevel` | string | No | — | `high`, `medium`, `low` (doubao-seed3d only) |
| `--quality_override` | int | No | — | Polygon face count (doubao-seed3d only) |
| `--webhook` | string | No | — | Callback URL |
| `--extra` | string | No | — | JSON object string; merged into request body top-level |
| `--api_key` | string | No | from env | — |

### Model Selection

- `--image` provided → `i23d` (default: `hyper3d-rodin`)
- No `--image` → `t23d` (default: `hyper3d-rodin`)
- `doubao-seed3d` requires `--image`

### Success Output

```json
{
  "status": "pending",
  "taskid": "<taskid>",
  "model": "<model>",
  "queue_position": 0,
  "ai302_cost_request_ids": "<request-id or ''>",
  "request_id": "<request-id or null>",
  "request": {
    "method": "POST",
    "url": "/302/submit/hyper3d-rodin",
    "headers": {"Authorization": "[REDACTED]"},
    "params": {"webhook": "<webhook>"},
    "body": {"prompt": "<prompt>", "model": "<model>", "...": "..."}
  },
  "raw": {}
}
```

### Failure Output (exit code 1)

```json
{
  "status": "failed",
  "taskid": null,
  "error": "<message>",
  "retryable": false,
  "request": {
    "method": "POST",
    "url": "/302/submit/hyper3d-rodin",
    "headers": {"Authorization": "[REDACTED]"},
    "params": {"webhook": "<webhook>"},
    "body": {"prompt": "<prompt>", "model": "<model>", "...": "..."}
  }
}
```

### Examples

```bash
# Text-to-3D
302ai 3d create --prompt "a cute cat" --model hyper3d-rodin

# Image-to-3D
302ai 3d create --image "https://example.com/cat.png" --model hyper3d-rodin

# With options
302ai 3d create --image "https://example.com/cat.png" --fileformat glb --material PBR
```

## 3d fetch

Query task status and result.

```bash
302ai 3d fetch <taskid> [--short] [--model <model>] [--api_key <key>]
```

- `taskid`: Task ID (positional, required)
- `--model`: Optional; auto-read from local task snapshot if omitted
- `--short`: Output only `status`, `taskid`, `result_url` (if ready)
- `--api_key`: Optional override

### Success Output (full)

```json
{
  "status": "processing|completed",
  "taskid": "<taskid>",
  "result_url": "<glb_url|null>",
  "request": {
    "method": "GET",
    "url": "/302/submit/hyper3d-rodin?request_id=<taskid>",
    "headers": {"Authorization": "[REDACTED]"},
    "params": null,
    "body": null
  },
  "raw": {}
}
```

### Success Output (--short)

```json
{"status":"completed","taskid":"<taskid>","result_url":"<glb_url>"}
```

### Failure Output (exit code 1)

```json
{"status":"failed","taskid":"<taskid>","error":"<message>","retryable":false}
```

### Example

```bash
302ai 3d fetch task_abc123 --short
```

## Typical Async Workflow

```bash
# 1. Create task
302ai 3d create --prompt "a cute cat" --model hyper3d-rodin
# Returns: {"status":"pending","taskid":"task_abc123","ai302_cost_request_ids":"...",...}

# 2. Poll until completed
302ai 3d fetch task_abc123 --short
# Returns: {"status":"pending","taskid":"task_abc123"}
# ... wait ...
# Returns: {"status":"completed","taskid":"task_abc123","result_url":"https://..."}

# 3. Query cost (optional)
302ai record get <request_id>
```

## Notes

- `ai302_cost_request_ids` / `request_id`: use with `record get` to query cost details
- `result_url`: points to the generated 3D model file (e.g. `.glb`)
- `retryable`: boolean indicating if error can be retried (in error output)
- Task model is auto-recorded at create time; `fetch` reads it from local snapshot so `--model` is rarely needed
